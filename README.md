# 공공조달 수요 기반 창업 아이템·입지 추천 서비스

공공조달 입찰공고를 지역별 공공수요 신호로 해석해,  
예비창업자와 창업상담가가 **어느 지역에서 어떤 업종이 유망한지** 탐색할 수 있도록 돕는 데이터 분석 서비스입니다.

핵심은 단순히 "많이 발주된 품목"을 그대로 추천하는 것이 아니라,  
**공공수요·진입장벽·데이터 신뢰도를 함께 고려해 창업 탐색의 참고 자료**를 제공하는 것입니다.

> 2026 공공조달데이터·AI 활용 창업경진대회 출품작 (주최: 조달청)

---

## 왜 이 프로젝트인가

물류창고 현장에서 10년간 재고 불균형 문제를 다뤘습니다.  
악성재고는 대부분 수요를 잘못 읽은 결과였습니다.  
창업도 비슷합니다. 수요가 없는 지역, 맞지 않는 품목으로 시작하면 재고 부담과 폐업으로 이어질 수 있습니다.

공공조달 데이터는 공공기관의 실제 구매 이력입니다.  
이 데이터를 창업 전 단계에서 지역별 수요 패턴으로 해석하면,  
조금 더 근거 있는 아이템·입지 선택이 가능하다고 생각했습니다.

이 프로젝트는 바로 그 지점에서 출발했습니다.

---

## 이 서비스가 하는 것

- 조달청 나라장터 입찰공고를 자치구별로 수집합니다.
- 기관명 → `agency_type`, 공고명 → `item_category_detail` 두 가지 분류를 생성합니다.
- 공고수·금액·최근성을 결합한 `opportunity_score`를 산출합니다.
- 인구 1만명당 공고수(`bids_per_10k_population`)로 구별 규모 편향을 보정합니다.
- 행안부 연령 인구 데이터로 품목군별 소비층 적합도(`consumer_fit_score`)를 계산합니다.
- 소상공인 상가정보로 업종별 경쟁 포화도(`stores_per_10k`)를 측정합니다.
- 규제 업종과 데이터 부족 카테고리를 추천에서 자동 분리합니다.
- 추천 항목에 한해 Gemini 기반 공공수요 해석 문장을 생성합니다.

## 이 서비스가 하지 않는 것

- 창업 성공 여부를 예측하지 않습니다.
- 민간 소비수요(상권, 소비자 데이터)를 직접 반영하지 않습니다.
- `opportunity_score`는 공공수요 참고 지표이며, 창업 성공 예측값이 아닙니다.
- AI 설명 문장은 공고 수·금액·공공수요 점수를 바탕으로 한 데이터 해석이며, 투자 판단 근거가 아닙니다.

## 데이터 파이프라인

```text
조달청 입찰공고 API
  → 자치구별 공고 수집 (build_seoul_sample.py)
  → 기관명/공고명 분류 (classify_agency.py)
      기관명 → agency_type (8개 유형)
      공고명 → item_category_detail (18개 세부 유형, 기타/미분류 4.9%)
  → 자치구 × 품목군 매트릭스 생성 (build_opportunity_matrix.py)
  → opportunity_score 산출 (공고수 50% + 금액 30% + 최근성 20%)
  → 추천 정책 적용 (recommendation_flag: 추천 / 제외 / 데이터부족)
  → 인구 보정 피처 추가 (build_features.py)
  → Gemini 설명 문장 생성 (gemini_client.py, 추천 항목만)
  → Streamlit 대시보드 (streamlit_review.py)

행안부 연령별 인구 API
  → 자치구 × 연령대별 인구 수집 (population_age_api.py)
  → 품목군 소비층 적합도 산출 (build_consumer_fit.py)
      consumer_fit_score = 주소비층 연령 비중 (min-max 정규화)

소상공인 상가정보 API
  → 자치구 × 업종별 점포수 수집 (store_api.py)
  → 경쟁 포화도 산출 (build_competition_matrix.py)
      stores_per_10k = 인구 1만명당 점포수
```

## 주요 지표

| 지표 | 계산 방식 | 의미 |
|---|---|---|
| `opportunity_score` | 공고수(50%) + 금액(30%) + 최근성(20%) × 100 | 해당 구·품목군의 공공수요 종합 매력도 |
| `recommendation_flag` | 추천 / 제외 / 데이터부족 | 추천 가능 여부 정책 판단 |
| `bids_per_10k_population` | 공고수 ÷ (인구 / 10,000) | 인구 규모 편향을 보정한 수요 밀도 |
| `consumer_fit_score` | 주소비층 연령 비중 min-max 정규화 | 인구 구성 기반 소비층 적합도 |
| `stores_per_10k` | 점포수 ÷ (인구 / 10,000) | 업종별 경쟁 포화도 |
| `avg_lead_time_days` | 개찰일 - 공고일 (평균) | 수요 집행 속도 참고 지표 |

`opportunity_score` 가중치는 **공고 수가 수요 신호의 가장 직접적인 지표**라는 도메인 판단을 반영해 count에 가장 높은 비중을 두었습니다.  
금액은 규모 판단 보조 지표, 최근성은 수요 지속성 확인용으로 사용합니다.

| 플래그 | 조건 | 처리 |
|---|---|---|
| 추천 | 일반 항목 (10건 이상) | 점수 노출 + AI 설명 생성 |
| 제외 | 폐기물/환경 · 건설/공사 · 기타/미분류 | TOP3 미노출, 정적 경고 문구 |
| 데이터부족 | 공고 건수 10건 미만 | TOP3 미노출, 정적 안내 문구 |

규제 업종이나 진입장벽이 높은 영역, 표본 수가 적은 카테고리를 분리해  
단순 수요량이 많다는 이유만으로 상위 추천에 노출되지 않도록 설계했습니다.

| 공공기관/재단 | 강남구도시관리공단, 영등포구미래교육재단 |
| 의료기관 | 강남구보건소, OO병원 |
| 어린이집/보육 | 서울시 OO어린이집, 구립어린이집 |
| 학교 | OO초등학교, OO중학교 |
| 복지시설 | OO노인복지관, OO장애인복지원 |
| 문화/체육시설 | OO도서관, OO문화센터, OO체육관 |
| 기타/미분류 | — |

현재 표본은 행정기관 비중이 높아,

### 2) `item_category_detail` (세부 발주 유형, 18종)

공고명 텍스트 기반 규칙 분류입니다.  
현재 분류율은 **95.1%**, `기타/미분류` 비율은 **4.9%**입니다.

청소/환경미화, 방역/소독, 폐기물/환경, 급식/식자재, IT장비/전산,  
사무용품/소모품, 시설유지보수, 교육물품/교구, 의료/복지용품,  
행사/운영용역, 조경/녹지관리, 급수/전기/설비, 차량/운송,  
경비/보안, 인쇄/홍보물, 건설/공사, 보험/금융, 전문용역/컨설팅

## 현재 한계

### 1) 표본 편향
서울 7개 구만 수집된 상태라 전체 서울 대표성은 제한적입니다.  
강남구 비중이 높아 업무지구 특성이 결과에 반영되어 있습니다.

### 2) 소표본 카테고리
경비/보안(9건), 의료/복지용품(6건), 방역/소독(7건), 급식/식자재(4건)는

서울 25개 구 확장 시 보완 가능성이 있습니다.

### 3) 공공수요만 반영
현재 버전은 공공조달 데이터 중심이므로,  
민간 소비수요·상권 흐름까지 반영한 추천은 아닙니다.

### 4) Gemini 모델 preview 단계
현재 사용 모델(`gemini-3.1-flash-lite-preview`)은 preview 단계입니다.  
API 실패 시 수치 기반 폴백 문장으로 자동 대체되며 앱은 정상 작동합니다.

## 현재 판단

서울 7개 자치구의 조달청 입찰공고 데이터(최근 2년, 1,396건)를 수집·분류하여  
자치구 × 품목군 단위 공공수요 기회 점수를 산출합니다.

진입장벽이 높은 규제 업종(건설/공사, 폐기물/환경)과 표본이 부족한 카테고리(10건 미만)는  
추천에서 자동 제외하며, 추천 항목에 한해 공공수요 특성을 설명하는 AI 문장을 생성합니다.

현재 상태는 **공공수요 기반 창업 추천 서비스 프로토타입**으로 시연 가능한 수준입니다.  
다음 단계는 서울 25개 구 확장과 발표자료 최신화이며,  
현재 7개 구 기준 결과는 프로토타입 범위 내에서 해석하는 것이 적절합니다.

## 메모

`opportunity_score`는 **창업 성공 점수**가 아니라  
**공공수요 참고 점수**입니다.

AI 설명은 공고 수·금액·공공수요 점수를 바탕으로 한 데이터 해석 문장이며,  
창업 성공을 예측하거나 보장하지 않습니다.

이 프로젝트의 목적은 정답을 단정하는 것이 아니라,  
예비창업자가 지역과 품목을 탐색할 때 참고할 수 있는  
**근거 있는 출발점**을 제공하는 것입니다.
