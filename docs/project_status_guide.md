# 프로젝트 중간점검 설명서

이 문서는 지금 프로젝트가 어디까지 왔는지, 어떤 데이터를 어떻게 뽑았는지, 어떤 파일을 보면 되는지 정리한 사용자용 설명서입니다.

## 1. 네가 처음 말한 문제

출발점은 물류창고에서 10년간 일하며 경험한 재고 불균형과 악성재고 문제였습니다.

핵심 문제의식은 다음과 같습니다.

- 악성재고는 단순히 남은 물건이 아니라 수요를 잘못 읽은 결과이다.
- 수요가 없는 곳에 물건이 쌓이면 보관비, 처리비, 폐기비가 생긴다.
- 창업도 비슷하게, 수요가 맞지 않는 지역과 품목을 선택하면 재고 부담과 손실이 생길 수 있다.
- 그래서 창업 전 단계에서 지역별·품목별 수요 신호를 참고할 수 있는 데이터가 있으면 좋겠다.

## 2. 지금 프로젝트의 방향

현재 방향은 다음입니다.

```text
공공조달 데이터를 활용해
서울 주요 자치구별 공공수요 품목을 분석하고,
예비창업자와 창업상담가가 참고할 수 있는
사업 아이템·입지 판단 근거를 만드는 프로젝트
```

중요한 점은 이 프로젝트가 `창업 성공 예측기`가 아니라는 것입니다.

현재 목표는 다음에 가깝습니다.

```text
창업상담가가 사람처럼 종합 판단할 때 참고할 수 있는
공공수요 기반 근거 자료 만들기
```

## 3. 지금 샘플링한 지역

전국을 한 번에 분석하지 않고, 우선 서울시 안에서도 특성이 뚜렷한 자치구를 샘플링했습니다.

| 자치구 | 가설/지역 이미지 |
| --- | --- |
| 관악구 | 청년, 자취생, 대학가 |
| 강남구 | 업무지구, 고소득, IT |
| 마포구 | 홍대, 관광, 문화콘텐츠 |
| 영등포구 | 오피스, 금융, 교통 |
| 성동구 | 성수, 브랜드, 라이프스타일 |
| 종로구 | 공공기관, 전통상권, 관광 |
| 송파구 | 가족주거, 대형상권, 복합문화 |

이번 실행 결과에서는 모든 구가 충분히 잡힌 것은 아닙니다.
조달 공고 데이터 안에 해당 자치구명이 들어간 공고만 추출되기 때문입니다.

## 4. 어떤 데이터를 가져왔나

현재 사용한 API는 다음입니다.

```text
조달청_나라장터 입찰공고정보서비스
```

현재 샘플 스크립트는 최근 180일을 30일 단위로 나누고, 각 기간마다 여러 페이지를 가져옵니다.

실행 파일:

```text
src/collect/build_seoul_sample.py
```

실행 명령:

```bash
python -m src.collect.build_seoul_sample
```

## 5. 데이터 처리 흐름

현재 처리 흐름은 다음입니다.

```text
1. 조달청 API에서 입찰공고 수집
2. 원천 CSV 저장
3. 공고명, 수요기관명, 금액, 공고일 정리
4. 서울 자치구명 추출
5. 공고명 기반 품목군 분류
6. 자치구 x 품목군 매트릭스 생성
7. 공고 수, 금액, 최근성으로 opportunity_score 계산
8. 표, 히트맵, 요약 리포트 저장
```

## 6. 주요 파일 설명

| 파일 | 역할 |

| `src/preprocess/clean_bid_data.py` | 원천 데이터를 정제하고, 자치구와 품목군을 붙임 |
| `src/features/build_opportunity_matrix.py` | 자치구 x 품목군 매트릭스와 추천 점수 계산 |
| `src/features/build_features.py` | 인구/세대/낙찰정보 결합용 feature 테이블 생성 |

## 8. 핵심 분석표 컬럼

`outputs/tables/seoul_opportunity_matrix.csv`가 현재 가장 중요한 표입니다.

| 컬럼 | 의미 |
| --- | --- |
| `district` | 서울 자치구 |
| `district_profile` | 사람이 이해하기 쉬운 지역 특성 라벨 |
| `item_category` | 공고명 기반으로 분류한 품목군 |
| `bid_count` | 해당 자치구·품목군 공고 수 |
| `amount_sum` | 해당 자치구·품목군 추정금액 합계 |
| `amount_mean` | 평균 추정금액 |
| `latest_posted_date` | 가장 최근 공고일 |
| `count_score` | 공고 수 정규화 점수 |
| `amount_score` | 금액 정규화 점수 |
| `recency_score` | 최근성 점수 |
| `opportunity_score` | 최종 참고 점수 |

## 9. 현재 점수 산식

초기 산식은 단순하게 만들었습니다.

```text
opportunity_score =
공고 수 점수 50%
+ 금액 규모 점수 30%
+ 최근성 점수 20%
```

주의할 점:

- 이 점수는 창업 성공 가능성이 아닙니다.
- 공공조달 데이터 안에서 눈에 띄는 지역·품목 조합을 찾기 위한 참고 지표입니다.
- 나중에 인구/세대수 데이터를 붙이면 `인구 대비 수요`, `세대수 대비 수요`로 더 발전시킬 수 있습니다.

## 10. 이번 샘플 결과 요약

최근 실행 결과에서는 다음과 같은 샘플이 나왔습니다.

| 자치구 | 품목군 예시 |
| --- | --- |
| 강남구 | 환경개선/생활민원, IT/소프트웨어, 시설관리/공사 |
| 마포구 | 창업/경영지원 |
| 영등포구 | 환경개선/생활민원, 금융/보험 |
| 종로구 | 도서/콘텐츠, 금융/보험 |

해석 예시:

- `강남구 -> IT/소프트웨어`는 업무지구/IT 이미지와 연결해 볼 수 있습니다.
- `마포구 -> 창업/경영지원`은 소상공인/창업지원 맥락으로 설명할 수 있습니다.
- `종로구 -> 도서/콘텐츠`는 문화재단, 도서관, 구정 홍보/콘텐츠와 연결해 볼 수 있습니다.

## 11. 현재 한계

지금 결과는 샘플입니다.

- 서울 전체 공고를 완전히 다 긁은 것이 아닙니다.
- 관악구, 성동구, 송파구는 이번 샘플에서 충분히 잡히지 않았습니다.
- 품목군 분류는 키워드 사전 기반이라 아직 거칠고 `기타`가 생길 수 있습니다.
- 주민등록 인구/세대 데이터는 아직 결합하지 않았습니다.
- 낙찰정보도 아직 결합하지 않았습니다.
