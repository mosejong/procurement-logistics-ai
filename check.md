# 조달 데이터 분류·추천 2차 검수 기록

> 검수 기준일: 2026-04-22
> 수집 범위: 서울 7개 구 · 최근 2년 · 1,396건
> 검수 대상: 분류기(item_category_detail), 추천 점수(opportunity_score), 추천 정책(recommendation_flag), Gemini 설명 연동

## 1. 2차 검수 목적

1차 검수 이후 아래 작업이 반영되었다.

- 수집 범위 확장: 180일 → 2년
- 데이터 규모: 549건 → 1,396건
- `item_category_detail` 분류기 보정 (기타/미분류 15.6% → 4.9%)
- `opportunity_score` 계산 기준을 구 분류에서 세부 분류 기준으로 전환
- 추천 제외 / 데이터부족 정책 추가
- Gemini 설명형 문장 생성 연동
- Streamlit UI 버그 수정 (AI 잔상, 0건 안내 문구 등)

이번 2차 검수의 목적은
**"분류기 개선이 실제 추천 결과에 연결되었는지"** 와
**"현재 상태로 시연 및 발표가 가능한지"** 를 확인하는 것이다.

## 4. 추천 구조 2차 검수 결과

### 구조적 문제 해결
1차 검수 당시 핵심 문제는
`item_category_detail` 분류가 개선되었음에도
`opportunity_score`가 여전히 구 분류(`item_category`) 기준으로 계산된다는 점이었다.

현재는 이 구조를 수정하여:
- `opportunity_score` 계산 기준을 **item_category_detail 기반**으로 전환
- 출력 컬럼명은 `item_category`로 유지해 downstream 호환성 확보
- `consumer_fit` 매핑도 세부 분류명 기준으로 정렬

즉, **분류 → 점수 → 추천 화면** 이 같은 기준으로 연결되도록 수정했다.

opportunity_score 공식: `count_score × 0.5 + amount_score × 0.3 + recency_score × 0.2`
가중치는 "공고 수가 수요 신호의 가장 직접적인 지표"라는 도메인 판단을 반영해 count에 절반 비중을 부여했다. 금액은 규모 판단 보조 지표, 최근성은 수요 지속성 확인용이다. 세 항목 모두 0~1 min-max 정규화 후 합산해 0~100점으로 환산한다.

### recommendation_flag 정책

| 플래그 | 조건 | 처리 |
|---|---|---|
| 추천 | 일반 항목 | 점수 노출 + AI 설명 생성 |
| 제외 | 폐기물/환경·건설/공사·기타/미분류 | TOP3 미노출, 정적 경고 문구 |
| 데이터부족 | 공고 건수 10건 미만 | TOP3 미노출, 정적 안내 문구 |

규제 업종(건설/공사·폐기물/환경)과 표본 부족 카테고리(10건 미만)를 추천에서 분리했다.
기준은 `EXCLUDE_CATEGORIES`와 `MIN_BID_COUNT_FOR_RECOMMENDATION` 상수로 코드에 고정되어 재현 가능하다.

## 5. Gemini 연동 검수 결과

### 연동 구조
- 모델: `gemini-3.1-flash-lite-preview`
- 대상: `recommendation_flag == 추천` 항목만 API 호출
- 제외 / 데이터부족 항목은 API 미호출, 정적 문구 반환

### 동작 정책
| 상황 | 동작 |
|---|---|
| 추천 항목 | Gemini 설명형 문장 생성 |
| 제외 항목 | 정적 경고 문구 반환 |
| 데이터부족 항목 | 정적 안내 문구 반환 |
| 503 과부하 | 최대 3회 재시도 (2초 간격) |
| API 실패 | 수치 기반 폴백 문장 반환 |
| API 키 없음 | 폴백 문장 반환 (앱은 정상 동작) |

### 역할 명확화
Gemini는 현재 **공공수요 해석 문장 생성기** 로 연결되어 있다.

생성 문장의 목적:
- 공고 수, 금액 규모, 공공수요 점수, 소비층 적합도를 바탕으로 **설명형 문장** 생성
- 창업 성공 예측, 투자 판단, 확정적 추천 표현 금지
- 시스템 프롬프트에 "유망합니다", "추천합니다", "좋은 기회입니다" 금지 규칙 명시

이 방향은 현재 프로젝트 목적과 일치한다.

| 카테고리 | 7개 구 2년치 건수 | 비고 |

공공조달 수요 신호를 기준으로 한 참고 지표임을 사용자에게 명확히 안내해야 한다.

## 10. 1차 → 2차 종합 비교

1차 검수의 핵심 과제는
"분류기를 고도화해도 추천 결과에 반영되지 않는다"는 구조적 단절이었다.

2차 검수에서는 이 문제를 해결했고,
**분류 → 점수 → 정책 → 설명 문장** 이 하나의 흐름으로 연결되었다.

현재 기준으로는 Gemini 연동을 포함한 데모 시연이 가능한 상태이며,
다음 개선은 기능 추가보다 **범위 확장과 발표 완성도 보강** 에 가깝다.
